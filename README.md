### Hello.

I'm James. I build things at the intersection of systems engineering, machine learning, and
developer tooling — currently as a Senior Product Engineer at
[North](https://www.north.com), a payments platform.

I've been writing code professionally for 15 years: eBay (ML infrastructure at petabyte
scale), Property Meld (first engineering hire, scaled to 70 people), and now North (Rust
API gateway, multi-agent RAG platform).

These days I'm most interested in:

- **AI-augmented development** — how LLMs change *how* we build software, not just what we
  build. I use Copilot, Claude, and Cursor daily. I think about this a lot.
- **Systems programming** — Rust, Firecracker microVMs, container runtimes, distributed
  systems. I built [zypi](https://github.com/allen-munsch/zypi) because I wanted to
  understand what happens under the hood when you `docker run` something.
- **The craft of engineering** — not just shipping features, but how teams work, how
  decisions get made, how tools shape thinking. I write about this at
  [jamesmunsch.com/words](https://jamesmunsch.com/words).

#### Some things I've built

**[zypi](https://github.com/allen-munsch/zypi)** &middot; A Firecracker microVM container
runtime in Elixir. Converts Docker images to root filesystems, manages VM lifecycle,
sub-second boot times. [Writeup here.](https://jamesmunsch.com/words/2026-01-18-zypi-run--zypi-go.html)

**[mosaic](https://github.com/allen-munsch/mosaic)** &middot; A federated semantic query
engine built with Elixir and SQLite. Think "what if SQLite had a vector search layer and
could talk to other instances?"

**[autocrud](https://github.com/allen-munsch/autocrud)** &middot; Any SQLite database,
instant REST API. 120 lines of bash, no config, no schemas, no routes.
[Writeup here.](https://jamesmunsch.com/words/2026-05-05-things-you-can-fit-in-a-single-shell-script.html)

**[nest.shell](https://github.com/allen-munsch/nest.shell)** &middot; A NestJS parody
built entirely in bash. File-system routing, guards, pipes, controllers, a CLI, 43 passing
tests — all under 700 lines.

**[SQL-to-Redis transpiler](https://jamesmunsch.com/words/2025-03-20-i-heard-you-wanted-sql-in-your-nosql.html)**
&middot; A weekend project. Parse SQL, transpile to Redis commands. Not production-ready,
but the architecture holds up.
